# 旧版 IPGW 功能清单分析

基于对旧版 `d:\project\Go\ipgw\pkg\cmd` 目录的分析，旧版客户端提供了丰富的子命令，全面覆盖了日常测速、账号管理及客户端维护功能。

以下是旧版 `IPGW` CLI 的完备功能清单（这些将作为我们 Meta 系列新架构必须包含的特性）：

## 核心网络控制功能
1. **`login` (登录操作)**
   *   提供用户名密码直连登录网关 (`-u` 与 `-p` 单次覆盖)。
   *   支持缺省参数调用缓存账户快速登录。
2. **`logout` (设备注销)**
   *   主动切断当前外网设备的网络连接通道，释放 IP 租期。
3. **`test` (网络连通性测速)**
   *   专门封装的网络探测指令，可能由于探测网关分配状态或许可证是否生效（类比于新版本的 `TestInternet`，但旧版单独抽成了一个指令）。

## 账户与设备管理功能
4. **`info` (信息资产查询)**
   *   自动拉取并聚合深澜及计费后台数据。
   *   支持查看 **当前公网 IP**。
   *   支持查看 **账户余额** (Balance)。
   *   支持查看 **累计使用时间** 及时长账单。
   *   支持查看 **本月累计及剩余可用流量**。
5. **`kick` (后台全端强制下线)**
   *   针对校园网 "设备连接数量达上限" 痛点开发。
   *   能跳过网页自助服务大厅，通过 CLI 向计费中枢发送强制移除其它闲置设备的 POST 踢人指令，极大简化了顶号成本。

## 系统与持久化维护功能
6. **`config` (全生命周期设置管理)**
   *   支持终端交互方式修改账号(`account`)与基础环境参数。
   *   实现持久化缓存，达到“录入一次，免密无限次”的效果。
7. **`update` (版本更新)**
   *   内置升级器，支持比对远端 GitHub Release 进行拉取升级。
8. **`version` (版本打印)**
   *   输出当前软件版本与编译元数据。
