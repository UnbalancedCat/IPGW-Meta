# API 手册 - 进阶使用

目前的指令支持主要集中于基础网关的配置、解锁认证及状态检验。除下方文档记录的常用方法外，你也可以在任何指令后加上 `-h` 参阅内置全中文参数帮助。

---

## 历史版本 API 兼容性对照 (与旧版 `ipgw`)

本工具作为 `neucn/ipgw` 的重构升级版（IPGW-Meta），在设计之初就追求了旧版本的习惯平移。

### 核心兼容与增强指令
- `ipgw config account [add, del, set, show]`：完整兼容并替代原本的 `config account [add, del, set, list]`。新增了 `--default` 标志以及多账户共存管理。
- `ipgw login`：完整兼容。新版引入了 Smart Login，在多账号环境下会先查询当前设备的在线状态；若检测到已有其他账号在线，则自动注销旧会话后再登入新账号。
- `ipgw logout` / `ipgw test`：完整覆盖。`test` 增强了对全局代理（如 TUN 模式）环境的检测能力，通过网关 API 与公网探针双重验证，减少因代理或 DNS 缓存导致的误判。

### 新增进阶特性
- `ipgw kick -a`：向网关发送全端下线请求，踢出该账号下包含本机在内的所有在线设备。适用于设备连接数达到上限无法登录的情况。
- `ipgw update`：自动检测当前操作系统与架构，从 GitHub Releases 拉取对应的最新版本并完成原地替换升级。
- `--bind-ip <IP>` (全局参数)：指定出站流量绑定的源 IPv4 地址。适用于多网口设备（如同时连接有线和无线的主机），通过 `net.Dialer.LocalAddr` 实现，跨平台可用且无需管理员权限。
- `--config <路径>` (全局参数)：指定自定义配置文件路径，支持多环境隔离部署，读取和写入都在指定的文件上进行。
- `--verbose` / `-v` (全局参数)：启用 Debug 级别日志输出，打印完整的网络请求和响应细节，便于排查问题。
- `--version` / `-V` (全局参数)：打印当前版本号，同时会静默检查是否有新版本可用。

### 暂未迁移的功能
- 原有的详细账单查阅（`info -b/-r/-l` 等）目前未迁移。`info` 命令当前提供流量、时长、余额等概览信息。历史账单抓取功能后续将在迭代中补充。

## 1. 登录与多账号智能切换
如果你不想保存密码，可以直接使用即时账户临时登录：
```Bash
ipgw login -u <学号> -p <密码>
```

在已经通过 `config account add` 配置过多账户的情况下，只需通过 `-u` 指定目标账号，客户端会自动读取已存储的密码，并在检测到其他账号在线时自动完成切换：
```Bash
ipgw login -u <目标学号>
```

## 2. 账号与配置管理
工具提供了完整的本地配置保存功能，支持多账号管理：
```Bash
# 添加账户信息（首个添加的账户会自动设为默认账户）
ipgw config account add -u <学号> -p <密码>

# 修改已有账户的密码或将其设为默认账户
ipgw config account set -u <学号> [-p <新密码>] [--default]

# 展示所有已保存的账户（带 * 标识的为默认账户）
ipgw config account show

# 删除已保存的指定账户（会要求确认）
ipgw config account del -u <学号>
```

## 3. 注销网络连接
向深澜网关发送下线请求，断开当前设备的网络连接：
```Bash
ipgw logout
```

## 4. 网络状态检测
检查当前设备是否处于校园网环境、是否已登录认证，并测试外网连通性：
```Bash
ipgw test
```

## 5. 全端设备下线
向网关发送全端下线请求，踢出该账号下所有已登录设备（包含本机）。适用于设备连接数达到上限需要重置的场景：
```Bash
# 下线该账号下所有在网设备
ipgw kick -a
```

## 6. 在线更新
自动检测系统架构并从 GitHub Releases 拉取最新版本完成原地更新：
```Bash
ipgw update
```

## 7. 多网口绑定与探测
适用于拥有多个网卡的设备（如同时接入有线和 Wi-Fi，或通过 VPN 接入校园网的 NAS/服务器）。

### 探测所有网口
并发遍历系统中所有活跃的 IPv4 网卡，分别绑定对应 IP 向深澜网关探测，输出每个网口的校园网连接与认证状态：
```Bash
ipgw info --all
```

### 指定源 IP 操作
通过 `--bind-ip` 全局参数指定出站流量的源 IPv4 地址。程序会创建一个绑定了该 IP 的专属 HTTP 客户端来执行所有网关操作：
```Bash
# 通过指定网口登录
ipgw login --bind-ip 10.166.x.x

# 通过指定网口检测
ipgw test --bind-ip 10.166.x.x

# 通过指定网口登出
ipgw logout --bind-ip 10.166.x.x
```

> **提示：** 可以先用 `ipgw info --all` 查看各网口的校园网状态，找到处于校园网内的 IP 地址后再用 `--bind-ip` 指定。

## 8. Debug 日志
遇到登录异常或网络问题时，可以使用 `--verbose`（简写 `-v`）启用详细日志输出：

```Bash
ipgw login -u <学号> -p <密码> --verbose
```
verbose 模式会额外输出以下信息：
- CAS 动态参数的提取结果（`lt` / `execution`）。
- 登录过程中每一步 HTTP 重定向的 URL 与 Ticket 截获情况。
- 深澜 API 的原始响应内容（可帮助定位如 `ip_already_online_error` 等网关侧错误）。
