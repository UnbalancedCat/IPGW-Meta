# API 手册 - 进阶使用

目前的指令支持主要集中于基础网关的配置、解锁认证及状态检验。除下方文档记录的常用方法外，你也可以在任何指令后加上 `-h` 参阅内置全中文参数帮助。

---

## 历史版本 API 兼容性对照 (与旧版 `ipgw`)

本工具作为 `neucn/ipgw` 的重构升级版（IPGW-Meta），在设计之初就追求了旧版本的习惯平移。

### 核心兼容与增强指令
- `ipgw config account [add, del, set, show]`：完整兼容并替代原本的 `config account [add, del, set, list]`。新增了 `--default` 独立指针以及多账户同存的热重载切换护栏。
- `ipgw login`：完整兼容。新版引入了 **Smart Login 智能会话管理**，在多账户存储环境下，不再无脑盲目地请求底层；而是会先查询挂载账户一旦冲突会自动热挤出原用户，保证了脚本在静默自启动调用时具有极强的鲁棒性。
- `ipgw logout` / `ipgw test`：完整覆盖。`test` 被增强到了应对复杂的全局代理（如 TUN 模式）下直连测算模型，可以防御特定 DNS 劫持导致的假阳性网关误判。

### 新增进阶特性
- `ipgw kick -a`：强制从云端抹除该账户名下的全部授权登录点（踢出包含本机在内的所有在线终端），强硬破解由于网关限挂机器（比如设备上限仅限 2 台）导致的“无法登录”连携卡死局面。
- `ipgw update` [原生跨平台自更新]：无需手动下载覆盖！输入该命令，程序将全自动拉取 GitHub Releases 中的最新对应系统架构发行包，并使用黑科技穿透运行时文件锁，为您执行**原地无痛升级覆盖**。
- `--config <路径>` (全局参数)：为服务化脚本分发而生！允许无视所有的操作系统默认配置找寻，强行读取或挂载隔离出一个在您指定的特定路径下的纯天然 YAML，配置解耦互不污染。
- `--verbose / -v` (全局参数)：重构了绝多数 CLI 只用来打印单纯 `-v` 版本的死板设定，赋予其超强的抓包反馈全链路打印权限，成为分析疑难杂症的神级信标。
- `--version / -V` (全局参数)：优雅地打印当前您的客户端全称版本型号。不仅如此，它在执行时还会**静默且极速地比对云端最新版本**，一旦发现更新就会立刻通过显眼的提示通知你：请运行 `ipgw update` 升级！

### 暂时精简的高级查询目标集
- 原有的深层明细账单查阅（`info -b/-r/-l` 等）：为了保证核心通讯流组件在主干路的长效平稳并反哺对抗新的 WAF 等特征变化，`IPGW-Meta` 暂时将 `info` 命令里的深入“老账本账单历史”查阅爬虫进行降级剥离。目前它是一套极度清爽且高效宏观的资源使用 Dashboard 仪表盘。（历史账单抓取后续将被补充扩展在迭代路线中！）

## 1. 登录与多账号智能切换
如果你不想保存密码，可以直接使用即时账户临时登录：
```Bash
ipgw login -u <学号> -p <密码>
```

在已经通过 `config account add` 配置过多账户的情况下，只需通过 `-u` 指定目标账号，客户端将智能实现**免密读取**与**自动挤掉旧会话登入切换**（Smart Auto-Switch）：
```Bash
ipgw login -u <目标学号>
```

## 2. 账号与配置精细管理
工具提供了完整的本地配置保存功能，支持多账号无缝漫游管理：
```Bash
# 添加并配置账户信息（默认情况下首个添加的账户将作为缺省账户）
ipgw config account add -u <学号> -p <密码>

# 修改已有账户的密码或将其指定设置为缺省账户
ipgw config account set -u <学号> [-p <新密码>] [--default]

# 展示所有的账户存储状态（带 * 标识的为默认账户）
ipgw config account show

# 删除已保存的指定账户信息（需要二次确认）
ipgw config account del -u <学号>
```

## 3. 注销网络连通
立即向深澜系统的核心交换机通报下线，断开该账号关联的当前设备网络：
```Bash
ipgw logout
```

## 4. 网关状态强制全双向探测
不仅检查自己是否处于校园网环境返回当前网卡的真实内网 IP、验证登录授权状态，更能双向查验真正的外网连接情况（可以一定程度检测因部分 TUN 接管网络与系统 DNS 缓存造成的网络误判）：
```Bash
ipgw test
```

## 5. 设备极速管理与全端强行下线踢出
安全地向网关发送全端踢出请求。目前为了应对网关的反爬频次检查与限制机制，稳定提供一键下线账户所有已登录终端设备的硬核强踢指令。非常适合在到达设备连接数上限需要“洗牌”重置时使用：
```Bash
# 强制下线该账号下所有在网设备（包含当前的机器）
ipgw kick -a
```

## 6. 原地热更新自我进化

想要获得最新特性或错误修补？现在早已不需要繁琐地重新打开下载页面！借由内置的强大运行时置换链（Self-Update），只需轻轻一敲：
```Bash
ipgw update
```
它能完美识别你的底层操作系统与指令架构（哪怕是 `linux-arm` 或 `darwin`），自动从远端抓取发行包解压，并完成**极其惊叹地运行时自我更替**。全程不报错，无进程独占干扰！

## 7. 日志抓包溯源
遭遇了意料之外的阻断？账户资金异常需要抓包提示排雷？你随时可以使用 `--verbose`（简写 `-v`）全局标记启用极详尽的底层网络跟踪审计：

```Bash
ipgw login -u <学号> -p <密码> --verbose
```
**Debug 模式将无视遮盖，强力穿透并向您直观展现的极客细节：**
- 初始化底层网路的动态握手加密细节。
- 拉取到的每一次反爬虫阻断特征码与安全令牌的换取结果 (`lt/execution`)。
- CAS 统一认证中枢派发的每一步重定向 Cookie 及授权 Token 流。
- API 服务控制层下发的**完全纯天然原始JSON**报错信息（能够帮你瞬间定位如：`"ip_already_online_error"` 超限终端冲突、服务区无权限 等棘手底层问题）。
